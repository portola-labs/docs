## List Orders

Retrieve a paginated list of orders.

```bash
GET /v1/orders
```

### Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | integer | Page number (default: 1) |
| `pageSize` | integer | Results per page (default: 20, max: 100) |
| `status` | string | Filter by status |
| `customerId` | string | Filter by customer |
| `startDate` | string | Filter by order date (ISO 8601) |
| `endDate` | string | Filter by order date (ISO 8601) |

## Create Order

Create a new sales order.

```bash
POST /v1/orders
```

### Request Body

```json
{
  "customerId": "cust_xyz789",
  "shipToId": "branch_def456",
  "poNumber": "PO-2024-002",
  "orderDate": "2024-01-20",
  "lineItems": [
    {
      "productVariantId": "var_def456",
      "quantity": 20
    }
  ],
  "notes": "Customer requested blue packaging"
}
```

### Response

```json
{
  "data": {
    "id": "order_new123",
    "status": "draft",
    ...
  }
}
```

## Update Order

Update an existing order.

```bash
PATCH /v1/orders/:id
```

### Request Body

```json
{
  "status": "confirmed",
  "notes": "Updated notes"
}
```

## Update Order Status

Change an order's status.

```bash
POST /v1/orders/:id/status
```

### Request Body

```json
{
  "status": "shipped",
  "trackingNumber": "1Z999AA10123456784"
}
```

### Valid Status Transitions

| From | To |
|------|-----|
| draft | confirmed, cancelled |
| confirmed | processing, cancelled |
| processing | shipped, cancelled |
| shipped | delivered |

## Cancel Order

Cancel an order.

```bash
POST /v1/orders/:id/cancel
```

### Request Body

```json
{
  "reason": "Customer requested cancellation"
}
```

<Warning>
Orders that have been invoiced or shipped may have restrictions on cancellation.
</Warning>
