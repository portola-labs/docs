The Buddy API allows you to programmatically access and manage your data, including customers, products, orders, and more.

<Note>
API access requires a paid plan. Contact sales to enable API access for your organization.
</Note>

## Base URL

All API requests should be made to:

```
https://api.usebuddy.io/v1
```

## Authentication

All API endpoints require authentication using API keys.

```bash
curl https://api.usebuddy.io/v1/customers \
  -H "Authorization: Bearer YOUR_API_KEY"
```

See [Authentication](/api-reference/authentication) for details on generating and managing API keys.

## Response Format

All responses are returned in JSON format:

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

## Error Handling

Errors return appropriate HTTP status codes with details:

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Customer ID is required"
  }
}
```

| Status Code | Meaning |
|-------------|---------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request |
| 401 | Unauthorized |
| 404 | Not Found |
| 429 | Rate Limited |
| 500 | Server Error |

## Rate Limits

API requests are rate limited based on your plan:

| Plan | Requests per Minute |
|------|---------------------|
| Starter | 60 |
| Professional | 300 |
| Enterprise | Unlimited |
